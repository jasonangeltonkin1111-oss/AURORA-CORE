## Folder Governance
- **Purpose:** hold structural architecture contracts and blueprint routing.
- **What belongs here:** runtime-owner/layer/route structure contracts.
- **What must not belong here:** control-run law duplication, implementation code, runtime proof claims.
- **Authority boundary:** blueprints define architecture structure; control governs workflow; source files define implementation truth.
- **Update rules:** update index when blueprint files are added/retired or scaffold status changes.
- **No-go rules:** do not use blueprint text alone to claim trading, permission, or runtime proof.
- **Relationship to master index:** this folder is mapped from `control/02_MASTER_REPO_FILE_INDEX.md`.
